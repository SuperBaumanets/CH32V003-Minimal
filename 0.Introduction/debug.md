## Отладка кода на CH32V003 через программатор WCH-LinkE

### Программное обеспечение

- **OpenOCD** с поддержкой WCH (версия `openocd-wch`).  
  Скачать можно с официального репозитория (см. **пункт 5** в `introduction.md`)

- **Telnet-клиент** (встроен в большинство ОС)

- **Прошивка** (`.elf` или `.bin` файл), скомпилированная для CH32V003.

### Подключение программатора

Соедините программатор WCH-LinkE с целевым устройством по интерфейсу SWD:

| WCH-LinkE | CH32V003 |
|-----------|----------|
| SWDIO     | SWDIO    |
| SWCLK     | SWCLK    |
| GND       | GND      |


### Запуск OpenOCD

1. Откройте первый терминал.  
   Перейдите в директорию, где находится `openocd-wch` (или добавьте её в `PATH`, если еще не сделали этого).

2. Запустите OpenOCD с указанием конфигурационного файла для WCH-RISC-V.  
   Команда имеет вид:

   ```bash
   openocd-wch -f </путь/до/wch-riscv.cfg>
   ```

   Пример:

   ```bash
   openocd-wch -f ~/Program/OpenOCD-wlink/OpenOCD/bin/wch-riscv.cfg
   ```

   **Оставьте этот терминал открытым** – он выполняет роль сервера отладки.

### Подключение через Telnet

1. Откройте **второй** терминал.

2. Подключитесь к отладочному серверу OpenOCD по порту 4444:

   ```bash
   telnet localhost 4444
   ```

   Если подключение успешно, вы увидите приглашение OpenOCD (обычно `> `).

### Основные команды OpenOCD

Ниже приведены основные команды.

#### 1. Загрузка прошивки

Для загрузки образа в память микроконтроллера используется команда:

```
load_image <путь_к_файлу> <адрес_загрузки>
```

- `<путь_к_файлу>` – путь до файла прошивки (`.elf` или `.bin`).
- `<адрес_загрузки>` – адрес во флеш-памяти, куда будет записан образ. Для CH32V003 стартовый адрес обычно `0x08000000`.

Пример:

```
load_image ./firmware.elf 0x08000000
```

Если файл имеет формат `.elf`, адрес загрузки можно не указывать – OpenOCD сам определит его из ELF-секций. Но для надёжности лучше указать явно.

#### 2. Запуск выполнения

После загрузки прошивки можно запустить выполнение с заданного адреса:

```
resume <адрес>
```

Пример (запуск с начала флеш-памяти):

```
resume 0x08000000
```

Если опустить адрес, выполнение продолжится с текущего места (например, с точки останова).

#### 3. Установка точки останова

Для остановки выполнения в определённом месте установите аппаратную точку останова:

```
bp <адрес> 2 hw
```

Параметры:
- `<адрес>` – адрес в памяти, где нужно остановиться.
- `hw` – указывает на аппаратную точку останова (hardware breakpoint).

Пример:

```
bp 0x08000004 2 hw
```

Это установит точку останова на адресе `0x08000004`.

#### 4. Просмотр и очистка точек останова

- Просмотреть текущие точки останова можно командой:
  ```
  bp
  ```

- Удалить точку останова:
  ```
  rbp <номер_точки>
  ```
  Номер точки можно узнать из вывода команды `bp`.

#### 5. Пошаговое выполнение

- Выполнить одну инструкцию (шаг):
  ```
  step
  ```

- Выполнить одну инструкцию с возможностью входа в подпрограммы:
  ```
  step [адрес]
  ```

#### 6. Остановка выполнения

- Приостановить выполняющуюся программу:
  ```
  halt
  ```

### 7. Чтение/запись регистров и памяти

- Прочитать память по адресу:
  ```
  mdw <адрес> [количество]
  ```
  Пример: `mdw 0x08000000 4` – прочитать 4 слова по адресу 0x08000000.

- Записать значение в память:
  ```
  mww <адрес> <значение>
  ```

### 8. Выход из отладки

- Завершить telnet-сессию:
  ```
  exit
  ```

- Остановить OpenOCD можно нажатием `Ctrl+C` в первом терминале.
